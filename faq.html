Bagaimana Mekanisme Otomasi Depstore Bekerja?

Sistem ini merupakan jembatan integrasi antara dashboard user dengan Application Programming Interface (API) milik Pterodactyl. Ketika Anda mengisi formulir deployment, backend kami melakukan validasi ketersediaan resource pada node yang ditunjuk. Jika valid, sistem akan mengirimkan payload JSON berisi instruksi pembuatan user baru, pembuatan database, dan alokasi port secara bersamaan. Seluruh proses ini berjalan di latar belakang tanpa memerlukan intervensi administratif manusia secara langsung.

Mengapa Deployment Saya Mengalami Kegagalan (Error)?

Kegagalan deployment dapat disebabkan oleh berbagai faktor teknis yang kompleks. Pertama, Conflict Data; jika email atau username yang Anda masukkan sudah pernah terdaftar di database panel utama, sistem akan menolak permintaan baru untuk mencegah duplikasi data. Kedua, Resource Exhaustion; setiap node memiliki limitasi fisik pada CPU dan RAM. Jika node mencapai kapasitas 100%, API akan mengembalikan kode error 500. Ketiga, masalah pada sistem daemon (Wings) yang mungkin sedang dalam status offline atau maintenance.

Apakah RAM Unlimited Benar-Benar Tanpa Batas?

Dalam istilah teknis, paket Unlimited RAM berarti sistem tidak menetapkan batasan kuota pada level Docker Container. Namun, performa server Anda tetap bergantung pada total kapasitas fisik RAM yang tersedia pada Node. Penggunaan RAM yang terlalu masif (misal: kebocoran memori pada script Java) tetap dapat menyebabkan proses "Out of Memory" (OOM) Killer yang dilakukan secara otomatis oleh Kernel Linux untuk menjaga stabilitas node utama.

Bagaimana Cara Mengonfigurasi Server Setelah Selesai Deploy?

Setelah mendapatkan password dari dashboard, Anda harus segera melakukan login ke panel utama. Di sana, Anda memiliki kontrol penuh atas Startup Command, Environment Variables, dan File Manager. Anda dapat mengunggah file melalui SFTP atau File Manager bawaan browser. Jika Anda ingin mengganti versi bahasa pemrograman (misal dari Node.js 16 ke 20), Anda bisa mencarinya di menu "Settings" lalu ganti Docker Image sesuai dengan ketersediaan Egg yang ada.

Apakah Layanan Ini Memiliki Dukungan Teknis 24/7?

Depstore dioperasikan oleh komunitas Developer. Meskipun sistem berjalan secara otomatis selama 24 jam, dukungan teknis manusia dilakukan secara sukarela. Jika Anda menemukan bug pada sistem dashboard atau kegagalan API, sangat disarankan untuk melampirkan screenshot log error kepada pihak pengelola melalui saluran komunikasi resmi yang tersedia di profil developer.
